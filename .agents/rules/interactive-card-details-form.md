## [기술 스택 강제 지침]

### 1. React (컴포넌트 뼈대 조립 및 역할 분리)
- **컴포넌트 분리:** 하나의 파일에 모든 코드를 욱여넣지 마. 화면을 크게 **카드 디스플레이 영역(CardDisplay)**과 **입력 폼 영역(CardForm)**, 그리고 완료 후 보여질 **성공 화면(SuccessMessage)**으로 명확히 나누어 조립해 줘.
- **데이터 흐름 (단방향 바인딩):** 최상위 부모 컴포넌트(`App` 또는 `Main`)에서 카드 정보 상태(State)를 관리하고, 이 값을 '카드 디스플레이'와 '입력 폼'에 각각 Props로 내려주는 구조를 채택해. 폼에서 타이핑을 하면 부모의 상태가 바뀌고, 그 즉시 카드 디스플레이가 업데이트되는 원리야.

### 2. Tailwind CSS (정밀한 레이아웃 및 폼 UI 상태)
- **절대 좌표와 겹침 (Absolute Positioning):** 데스크톱 화면에서 앞면 카드와 뒷면 카드가 비스듬하게 겹쳐 있는 레이아웃은 `relative`와 `absolute`, 그리고 `z-index`를 활용하여 정밀하게 배치해 줘. 카드 배경의 화려한 그라데이션은 Tailwind의 `bg-gradient-to-br` 유틸리티를 활용해 이미지나 별도 CSS 없이 구현해.
- **반응형 레이아웃 (Mobile First):** 모바일(`default`)에서는 상단에 카드가 겹쳐서 배치되고 하단에 폼이 나오는 세로형(`flex-col`) 구조로, 데스크톱(`lg:`)에서는 좌측에 카드 세트, 우측에 폼이 위치하는 2단 가로형 구조(`flex-row`, `grid`)로 자연스럽게 전환되게 해 줘.
- **입력 폼 상태 (Active/Error):** 사용자가 인풋 창을 클릭했을 때(Focus) 테두리가 보라색 그라데이션으로 변하도록 `focus:border-purple-500` 등의 클래스를 적용해. 에러가 발생한 인풋은 테두리가 즉시 빨간색(`border-red-500`)으로 바뀌어야 해.

### 3. React State & 로직 (실시간 포맷팅 및 검증)
- **상태 구조:** 입력값(Values)과 에러 메시지(Errors)를 분리해서 관리해 줘. 
  예: `const [formData, setFormData] = useState({ name: '', number: '', month: '', year: '', cvc: '' })`
- **실시간 포맷팅 (카드 번호):** 카드 번호 입력 필드는 사용자가 숫자를 칠 때마다 **4자리마다 자동으로 공백(space)이 들어가도록**(`1234 5678...`) `onChange` 핸들러에서 정규식이나 문자열 조작 로직을 추가해 줘. 이 포맷팅된 값이 그대로 카드 디스플레이에 반영되어야 해.
- **빈 값 처리 (Placeholder):** 사용자가 아직 아무것도 입력하지 않았다면, 카드 디스플레이에는 'JANE APPLESEED', '0000 0000 0000 0000', '00/00', '000' 같은 기본 Placeholder 텍스트가 표시되도록 조건부 렌더링(`||` 연산자 활용)을 적용해 줘.
- **엄격한 폼 검증 (Validation):** 'Confirm' 버튼을 누를 때 다음 조건을 모두 검사해.
  1. **빈칸 확인:** 하나라도 비어있으면 "Can't be blank" 에러.
  2. **숫자 형식:** 카드 번호, 만료일, CVC에 문자가 들어가면 "Wrong format, numbers only" 에러.
  3. 에러가 발생한 필드 바로 아래에 빨간색 텍스트로 해당 에러 메시지를 띄워 줘.
- **화면 전환 (Submit Success):** 에러 없이 폼 제출이 완료되면 `isSubmitted` 상태를 `true`로 변경하고, 기존 입력 폼 컴포넌트 대신 체크마크가 있는 'Thank You!' 완료 화면 컴포넌트를 렌더링해 줘.

### 4. 접근성 (Accessibility & UX)
- **라벨과 인풋 연결:** 모든 `<input>` 태그는 반드시 고유한 `id`를 가지고, 짝지어진 `<label>`의 `htmlFor` 속성과 연결되어야 해. 시력 보호를 위해 라벨 텍스트를 클릭해도 인풋에 포커스가 가도록 해 줘.
- **에러 리더기 지원:** 에러 메시지가 나타나는 태그에는 `role="alert"`를 추가하고, 입력창에 에러가 났을 때는 `<input>`에 `aria-invalid="true"`와 `aria-describedby="에러메시지_ID"`를 부여해서 스크린 리더 사용자가 어떤 필드에서 왜 에러가 났는지 정확히 알 수 있게 해.
- **키패드 최적화:** 모바일 사용자를 위해 숫자만 입력해야 하는 카드 번호, 만료일, CVC 필드는 `<input type="text" inputMode="numeric" pattern="[0-9]*">`를 설정해서 모바일에서 숫자 키패드가 먼저 올라오게 하는 디테일을 챙겨 줘.
